# @dojocho/ui

The dojo web UI and A2A server. **MIT-licensed.** Built fresh — no
upstream code copied; everything below is original.

- **Backend transport:** [Hono](https://hono.dev) running inside
  [TanStack Start](https://tanstack.com/start) via
  [Nitro](https://nitro.unjs.io). The Hono app is the single source of
  truth for HTTP routes; TanStack Start file routes delegate to it via
  `app.fetch(request)`.
- **Agent protocol:** A2A 0.3.0, via the
  [`@a2a-js/sdk`](https://github.com/a2aproject/a2a-js) (Apache-2.0)
  imported as a runtime dependency. We wrote our own Hono adapters; we
  did not copy code from the SDK's Express bindings.
- **Frontend:** TanStack Start + React 19. Minimal UI shell for now.

## Why this stack

- One Hono app handles the whole backend. Same code in dev (`vite dev`),
  in `vite preview`, and in any Nitro preset (vercel, cloudflare,
  node-server). Hono is runtime-agnostic so we can target Bun/Node/Deno
  without rewriting routes.
- TanStack Start is already used by `apps/web`; we keep the React/Vite
  stack consistent across the monorepo.
- A2A is the external protocol of record (see `docs/ARCHITECTURE.md`).
  Using the official SDK gives us the spec types (`AgentCard`, `Task`,
  `Message`, `Part`, `JSONRPCResponse`) and the transport-agnostic
  `JsonRpcTransportHandler` for free.

## What's wired (v1)

- ✅ `GET /.well-known/agent-card.json` — A2A Agent Card discovery
- ✅ `POST /api/a2a/jsonrpc` — A2A JSON-RPC
- ✅ Non-streaming methods: `message/send`, `tasks/get`, `tasks/cancel`,
  `tasks/pushNotificationConfig/*`, `agent/authenticatedExtendedCard`
  (whatever the SDK's `JsonRpcTransportHandler` dispatches)
- ✅ Streaming methods (`message/stream`, `tasks/resubscribe`) — adapter
  serializes the SDK's `AsyncGenerator` as SSE. Marked as **disabled**
  in the agent card (`capabilities.streaming: false`) until verified
  against Nitro's response buffering.
- ✅ `EchoExecutor` AgentExecutor stub — replies "You said: …" so we can
  test the round trip without real harness adapters.
- ✅ `InMemoryTaskStore` — fine for v1; SQLite/persistent later.

## What's NOT wired (yet)

- ❌ Real broker — forwarding to opencode HTTP+SSE / gemini A2A /
  codex `app-server` / pi `--mode rpc`. Tracked in
  `docs/ARCHITECTURE.md`. Each adapter is its own AgentExecutor;
  `EchoExecutor` is a placeholder.
- ❌ Auth — JWT/master key (will lift the patterns from claude-a2a).
- ❌ Pairing/session binding — needed before exposing past localhost.
- ❌ React kata UI — Monaco editor, transcript pane, test results,
  approval modal. The `routes/index.tsx` page is a placeholder.
- ❌ Persistent task store — using in-memory.

## Running

```bash
# from apps/ui/ (or via dojo ui from anywhere)
pnpm install
pnpm dev                       # → http://localhost:4567

# smoke-test the agent card
curl http://localhost:4567/.well-known/agent-card.json

# smoke-test message/send (echo)
curl -X POST http://localhost:4567/api/a2a/jsonrpc \
  -H 'Content-Type: application/json' \
  -d '{"jsonrpc":"2.0","id":1,"method":"message/send",
       "params":{"message":{"messageId":"m1","role":"user",
                            "parts":[{"kind":"text","text":"hi"}]}}}'
```

Override the port with `PORT=… pnpm dev` (or `DOJO_UI_PORT` if launched
through `dojo ui`).
